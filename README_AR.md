# 🧘 التعافي المزدوج (Psycho-Kinetic Recovery) — LangGraph

معظم برامج التأهيل الرياضي تعتني بالجسد وتتجاهل العقل. هذا المشروع **يدمج الاثنين
ديناميكياً كل يوم**.

مبني بـ **LangGraph** كرسم بياني ذي حالة مستمرة: يُدخل اللاعب تقييمه اليومي (الألم
والمزاج)، فيعيد النظام كتابة **خطة الغد البدنية والنفسية** فوراً — مع تذكّر تطوّره عبر الأيام.

> 📄 شرح مفصّل بالعربية بصيغة PDF داخل مجلد `docs/`.
> 🌐 النسخة الإنجليزية في [README.md](README.md).

## ⚙️ كيف يعمل

```
      تقييم يومي (الألم، المزاج)   ← تدخّل المستخدم (human-in-the-loop)
                 │
                 ▼
         ┌──────────────┐
         │  عقدة القرار  │  ← قرار حتمي (adjust_plan)
         └──────┬───────┘
                ▼
         ┌──────────────┐
         │  وكيل بدني    │  ← يكتب خطة الجسد
         └──────┬───────┘
                ▼
         ┌──────────────┐
         │  وكيل نفسي    │  ← يكتب خطة العقل
         └──────┬───────┘
                ▼
     خطة الغد المزدوجة + حفظ التاريخ
```

- **الحالة تُحفظ** عبر الأيام بواسطة `MemorySaver`.
- **القرار حتمي**: الألم يقود الحِمل البدني، والمزاج يقود الدعم النفسي.
- **قاعدة سلامة صارمة**: ألم عالٍ + مزاج سيء = يوم راحة إجباري لا يتجاوزه الذكاء الاصطناعي.
- **مربوط بأساس ثابت (baseline)** فتعكس الخطة حالة اليوم لا خطأً متراكماً.

---

## 🛠️ التقنيات

- **LangGraph** — رسم بياني ذو حالة مع حفظ (checkpointing)
- **LangChain + OpenRouter** — الوصول إلى النماذج اللغوية
- منطق قرار حتمي في `science.py` معاد استخدامه من مشاريع التنبؤ بالإصابة

## 🚀 التشغيل

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env             # ضع مفتاح OpenRouter
python main.py                     # تقييم يومي تفاعلي (اكتب q للخروج)
```
